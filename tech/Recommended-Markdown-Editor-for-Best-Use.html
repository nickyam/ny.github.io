两款最好用的Markdown编辑器推荐

今天又看到有小伙伴在争论 Markdown 到底好不好用。

私以为，如果觉得不好用，那你本身就不是 Markdown 的受众。Markdown从设计之初就无法满足譬如页眉页脚页码，水印目录居中、标宋仿宋红头等等的功能需求。

Markdown 语言在 2004 由约翰·格鲁伯（英语：John Gruber）创建。Markdown 是一种轻量级标记语言，它允许人们使用易读易写的纯文本格式编写文档。它本质上就是个写Blog的语言，不是用来写公文搞排版的。Markdown可以使用网页编写的语言html的语法进行99%格式渲染。可以使用txt编辑器自己编写，最后进行渲染。它的优点就是极简主义，并且是博客网站的通用标记语言。

当然，这不是今天讨论的重点。自从用 GitHub Page 搭建个人博客开始，笔者就一直在用各种编辑器，有免费的、付费的、在线的、破解的。一通使用下来，反复对比下挑选了两款最好用的，今天推荐给大家！

Typora

下载地址：https://typora.io

  • 推荐指数：☆☆☆☆
  • 支持 Windows、OS X 和 Linux三种系统
  • 支持即时渲染技术，所见即所得，这也是与其他 Markdown 编辑器最显著的区别
  • 支持数学编辑，可与 Word 直接格式转换
  • 在 Pandoc 的支持下进行多种文档格式转换

但是它有一个缺点，收费（所以才给了四星）。所以请各位小伙伴自行考虑。

所以我要推荐第二款，也是我认为的神器，也是我现在每天在用的工具。

VScode

推荐指数：☆☆☆☆☆

程序猿一族们最常用的编辑器，而且它原生支持 Markdown 语法，非常丝滑，不过需要安装以下几个插件。

markdown all in one

将原生markdown预览的黑色背景改为白色
